## ADDED Requirements

### Requirement: Deterministic social image generation

The skill SHALL produce the social image using deterministic tooling (a
headless-browser screenshot), and SHALL NOT use an AI image model. Given the same
inputs it SHALL produce a byte-stable-enough image (same layout, text, and
dimensions) on repeated runs.

#### Scenario: No AI in the pipeline
- **WHEN** the skill generates a social image
- **THEN** it renders HTML/CSS and screenshots it with a headless browser, with no
  call to any AI/image-generation model or paid image API

#### Scenario: Repeatable output
- **WHEN** the skill is run twice with identical inputs
- **THEN** the produced image has the same dimensions and content (no randomness,
  no per-run AI variation)

### Requirement: Standard social image dimensions and output location

The generated image SHALL be 1200×630 (the Open Graph / Twitter summary-large
size) and SHALL be written to the project's static-served root as
`public/og-image.png` (or the project's equivalent deploy-root static folder).

#### Scenario: Correct size and path
- **WHEN** the skill generates the image for a project
- **THEN** the output is a 1200×630 PNG at the project's static root suitable for
  serving at `/og-image.png`

### Requirement: Generate from an existing landing hero

When a project has a landing page, the skill SHALL screenshot that landing's hero
region (a configurable CSS selector, with a sensible default) at the OG
dimensions.

#### Scenario: Landing with a hero
- **WHEN** the project has a landing page and a hero selector is available
- **THEN** the skill renders that landing and captures the hero region into the
  1200×630 social image

#### Scenario: Custom hero selector
- **WHEN** the caller supplies a hero selector
- **THEN** the skill captures that element instead of the default

### Requirement: Generate a throwaway hero for projects without a landing

When a project has no landing page, the skill SHALL render a **throwaway hero
HTML** — project name, tagline, and icon, styled with the GA ui-kit tokens — and
screenshot it. This throwaway hero MUST NOT be added to the project's deployed
output.

#### Scenario: Project without a landing (e.g. boids)
- **WHEN** the skill is asked to generate a social image for a project that has no
  landing page
- **THEN** it builds a hero HTML from the project's name/tagline/icon, renders it
  headlessly, screenshots it into the social image, and does not leave a deployed
  landing behind

#### Scenario: Hero uses the project's real assets
- **WHEN** the throwaway hero is rendered
- **THEN** it uses the project's icon and text and the GA ui-kit visual tokens so
  the image matches the house style

### Requirement: Reference the image in project metadata

The skill's output SHALL be referenced by the project via `og:image` and
`twitter:image` (absolute URL to the live site), so link unfurls and the
garutyunov.com crawler resolve it.

#### Scenario: OG tags wired
- **WHEN** the social image has been generated for a project
- **THEN** the project's HTML head includes `og:image` (and `twitter:image`)
  pointing at the served image URL

### Requirement: Pet-project-metadata requires a social image

The `pet-project-metadata` skill SHALL be updated so its required-metadata
contract and audit checklist include a social image generated by the
`social-image` skill and referenced in the project's OG tags.

#### Scenario: Metadata audit covers the social image
- **WHEN** a pet project is audited against `pet-project-metadata`
- **THEN** the checklist includes "has a 1200×630 social image (`og-image.png`)
  generated by the `social-image` skill and referenced via `og:image`"

### Requirement: Skill installed per-repo with the standard layout

The new skill SHALL follow this repo's skill layout: source in
`.agents/skills/social-image/`, a symlink at `.claude/skills/social-image`, and a
`skills-lock.json` entry — never a global install.

#### Scenario: Correct skill layout
- **WHEN** the skill is added
- **THEN** its files live under `.agents/skills/social-image/`, `.claude/skills/social-image`
  is a symlink to it, and no other agent directories are touched
