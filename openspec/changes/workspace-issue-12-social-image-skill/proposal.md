## Why

Pet projects need a social/OG image so links unfurl nicely on LinkedIn, Slack,
etc. — and garutyunov.com's card crawler already reads `og:image` off each
project's live page. Today there is no standard, deterministic way to produce
that image: `icon-generation` makes a square app icon via an AI model, but a
social image is a wide (1200×630) hero shot, and the owner wants it produced by
**deterministic tooling, not an AI model**. Projects that ship a landing page can
screenshot its hero; projects without one (e.g. boids) have nothing to shoot.

## What Changes

- Add a new **`social-image`** skill to this repo that deterministically renders a
  project's landing **hero** into a 1200×630 social image via a **headless-Chrome
  screenshot** (Playwright), no AI.
- Two modes:
  - **From an existing landing**: screenshot the hero section of the project's
    landing page (a configurable selector) at OG dimensions.
  - **No landing (e.g. boids)**: the skill renders a **throwaway hero HTML**
    (title, tagline, icon, styled with the GA ui-kit tokens/components) that is
    **never deployed** — it exists only to be screenshotted.
- Output a `public/og-image.png` (1200×630) the project references via
  `og:image` / `twitter:image`.
- **Update the `pet-project-metadata` skill**: add a required-metadata rule that
  every pet project MUST include such a social image, generated by this skill and
  referenced in its OG tags — extending the existing metadata contract.

## Capabilities

### New Capabilities
- `social-image-generation`: a deterministic (no-AI) skill that produces a
  1200×630 social image from a project's landing hero — screenshotting a real
  landing's hero, or a throwaway ui-kit-styled hero for projects without a
  landing — and the conventions for where it lives and how it's referenced.

## Impact

- **`.agents/skills/social-image/`**: new `SKILL.md` + `scripts/` (the headless
  render/screenshot script and the throwaway-hero HTML template), with the
  standard `.claude/skills/social-image` symlink and a `skills-lock.json` entry.
- **`.agents/skills/pet-project-metadata/SKILL.md`**: add the social-image
  requirement to the required-metadata table, conventions, and audit checklist;
  cross-link the new skill.
- **Tooling**: Playwright (headless Chromium) as the deterministic renderer,
  invoked via `npx` at skill-run time — no permanent heavy dependency added to
  any project repo.
- **Downstream pet projects**: each gains a `public/og-image.png` and OG tags;
  garutyunov.com's crawler picks it up automatically (no change there).
- **No secrets / no external paid API** (unlike `icon-generation`'s OpenRouter
  key) — fully local and reproducible.
